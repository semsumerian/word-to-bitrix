# Конвертер для Bitrix

Локальный MVP-инструмент `Word в HTML для учебных курсов` для автоматической подготовки HTML из Word-файлов справочника `Курсы обучения` для публикации в Bitrix Learning.

Инструмент нужен, чтобы сотрудники не переносили большие Word-документы в HTML вручную. Он читает Word-файл, применяет правила актуализации, формирует чистый HTML для поля `DETAIL_TEXT`, показывает предпросмотр, компактную статистику и подробности обработки.

## Для Кого

**Аналитик / инициатор актуализации**

- Загружает Word-файл.
- Проверяет предпросмотр.
- Смотрит, что текст с красной заливкой удален, а желтые и зеленые добавления остались без подсветки.
- Передает HTML техподдержке или вставляет его в Bitrix при наличии доступа.

**Техподдержка / администратор Bitrix**

- Запускает локальный сайт.
- Проверяет HTML, предпросмотр, предупреждения и подробности обработки при необходимости.
- Вставляет HTML в урок Bitrix Learning.
- При ошибках смотрит предупреждения и логику диагностики ниже.

## Что Делает Инструмент

- Принимает `.doc`, `.docx`.
- Для старых `.doc` использует LibreOffice и переводит документ во внутренний `.docx`.
- Читает структуру Word напрямую из OOXML, а не копирует грязный HTML из Word.
- Удаляет фрагменты с красной заливкой.
- Оставляет фрагменты, выделенные желтым или зеленым, но убирает саму подсветку.
- Сохраняет текст, у которого просто изменен цвет букв, включая красный цвет шрифта.
- Сохраняет зачеркнутый текст, если у него нет красной заливки.
- Удаляет строки таблиц с красной заливкой до пересчета объединенных ячеек.
- Пересчитывает `rowspan`, чтобы таблицы не ехали после удаления строк.
- Задает ширины таблиц через общую сетку колонок, чтобы объединенные ячейки не сдвигали границы и таблицы растягивались по ширине страницы.
- Центрирует содержимое ячеек по вертикали, если в Word не задано другое выравнивание.
- Добавляет перенос длинного текста внутри ячеек, чтобы предпросмотр и сайт не показывали текст поверх соседних колонок.
- Предупреждает о подозрительных пустых объединенных ячейках, датах с `rowspan` и датах, после которых в столбце срока идут пустые ячейки.
- Сохраняет таблицы, ссылки, жирный, курсив, подчеркивание, базовые цвета и выравнивание.
- Сохраняет автонумерацию Word, например `1.` и `2.`, если номер хранится не текстом, а как список Word.
- Если в Word есть изображения, вставляет шаблоны ссылок вида `__IMAGE_1_URL__`, чтобы сотрудник заменил их на реальные URL после загрузки картинок в Bitrix/Etalon.
- Не переносит Word-стили шрифта вроде `font-family`, `font-size`, `line-height` и `mso-font-*`.
- Формирует структурированный HTML с отступами и переносами строк.
- Показывает предпросмотр HTML на сайте.
- Показывает компактную статистику результата и сворачиваемые подробности по удаленным и добавленным фрагментам.
- Для CLI-запуска и диагностики генерирует JSON-отчет по удаленным и добавленным фрагментам.

## Главные Правила Обработки

| В Word | Что будет в HTML |
| --- | --- |
| Красная заливка / красная подсветка | Фрагмент удаляется |
| Красный цвет букв без заливки | Текст остается красным |
| Любой другой цвет букв | Текст остается |
| Зачеркнутый текст без красной заливки | Текст остается зачеркнутым |
| Желтая или зеленая заливка / выделение | Текст остается, подсветка удаляется |
| Word-стили шрифта: гарнитура, размер, межстрочный интервал | Не переносятся |
| Обычный текст | Текст остается |
| Таблицы | Структура сохраняется |
| Строки таблицы с красной заливкой | Строки удаляются, `rowspan` пересчитывается |
| Ссылки | Ссылки сохраняются |
| Изображения Word | Вставляется `<img src="__IMAGE_N_URL__">`, подпись остается отдельной строкой |
| Автонумерация Word | Номера вставляются в HTML как обычный текст |

## Что Видно После Обработки

На странице результата показываются:

- имя обработанного файла;
- основные действия: `Скопировать HTML`, `Скачать HTML`, `Новый файл`;
- компактная статистика: сколько фрагментов удалено, сколько добавлено, сколько таблиц найдено и сколько символов в HTML;
- предупреждения, если конвертер нашел изображения, подозрительные таблицы или другие ситуации для проверки;
- блок `HTML для Bitrix` с готовым кодом;
- блок `Предпросмотр`;
- сворачиваемый блок `Подробности обработки` со списком удаленных красных и добавленных фрагментов.

## Быстрый Запуск Сайта

Открыть терминал и выполнить:

```bash
cd "/Users/semen/Курсы обучения/word-to-bitrix"
python3 server.py
```

Открыть в браузере:

```text
http://127.0.0.1:8080
```

Если порт `8080` занят, сайт можно запустить на другом порту:


И открыть:

```text

```

Чтобы остановить сайт, в терминале нажать `Ctrl+C`.

Если код инструмента обновлялся, сайт нужно остановить и запустить заново.

## Сценарий Для Аналитика

1. Открыть `http://127.0.0.1:8080`.
2. Нажать выбор файла.
3. Загрузить Word-документ, например `Приложение 6 ККП_июнь.doc`.
4. Нажать `Подготовить HTML`.
5. Проверить компактную статистику результата.
6. Проверить, что фрагменты с красной заливкой исчезли.
7. Проверить, что желтые и зеленые фрагменты остались без подсветки.
8. Проверить таблицы: строки, объединенные ячейки, заголовки, нумерацию.
9. Проверить блок `Предпросмотр`.
10. При необходимости раскрыть `Подробности обработки` и посмотреть удаленные/добавленные фрагменты.
11. Если все корректно, нажать `Скопировать HTML`.
12. Передать HTML техподдержке или вставить в Bitrix при наличии доступа.

## Сценарий Для Техподдержки

1. Запустить сайт командой из раздела `Быстрый Запуск Сайта`.
2. Загрузить Word-файл через сайт.
3. Проверить компактную статистику результата.
4. Проверить предупреждения, если они есть.
5. Проверить предпросмотр.
6. При необходимости раскрыть `Подробности обработки`.
7. Скопировать HTML кнопкой `Скопировать HTML` или кнопкой `Скопировать` в блоке `HTML для Bitrix`.
8. Открыть нужный урок в Bitrix.
9. Вставить HTML в поле содержания урока.
10. Сохранить урок.
11. Открыть пользовательскую страницу и проверить отображение.

## Как Вставлять В Bitrix

В админке Bitrix используется модуль `learning`.

Обычно страница редактирования выглядит так:

```text
/bitrix/admin/learn_unilesson_edit.php
```

Порядок действий:

1. Открыть нужный урок.
2. Перейти на вкладку `Содержание`.
3. Выбрать источник содержимого `текстовое поле`.
4. Выбрать режим `HTML` или режим исходного HTML в визуальном редакторе.
5. Вставить HTML из поля `HTML для Bitrix`.
6. Сохранить урок.
7. Открыть страницу урока для сотрудника и проверить результат.

Основное поле Bitrix:

```text
DETAIL_TEXT
```

## Что Проверять Перед Публикацией

Минимальный чек-лист:

- Нет фрагментов с красной заливкой, которые должны были быть удалены.
- Текст, у которого просто красный цвет букв, не удален.
- Желтые и зеленые фрагменты остались в тексте.
- Желтая и зеленая подсветка не попали в итоговую публикацию.
- Таблицы не развалились.
- Таблицы подстраиваются под ширину страницы, а границы колонок не уезжают из-за объединенных ячеек.
- Текст не выходит за границы ячеек в предпросмотре.
- Короткие значения и даты визуально стоят по центру ячейки.
- Объединенные ячейки выглядят корректно.
- Если в HTML есть `__IMAGE_N_URL__`, все такие шаблоны заменены на реальные ссылки картинок из Bitrix/Etalon.
- В таблицах нет пустых строк от удаленных данных с красной заливкой.
- Нумерация Word сохранилась, например `1.`, `2.`.
- Ссылки открываются.
- В предупреждениях нет критичных проблем.
- Если есть сомнения по правкам, в блоке `Подробности обработки` совпадает ожидаемый список удаленных и добавленных фрагментов.
- Пользовательская страница Bitrix выглядит так же, как предпросмотр или ручной эталон.

## Предупреждения И Что Делать

**Фрагменты с красной заливкой не найдены**

Это не всегда ошибка. Такое бывает, если файл уже очищен вручную или в документе нет удаляемых фрагментов с красной заливкой.

**Добавленные фрагменты не найдены**

Это не всегда ошибка. Такое бывает, если документ не содержит новых данных с желтой или зеленой заливкой.

**В документе найдены изображения**

Конвертер не загружает картинки в Bitrix автоматически. В HTML будут вставлены шаблоны `__IMAGE_1_URL__`, `__IMAGE_2_URL__` и т.д. Нужно загрузить картинки в Bitrix/Etalon и заменить каждый шаблон на реальный URL.

**Системный конвертер отдал HTML с поврежденной структурой таблиц**

Нельзя публиковать результат без ручной проверки. Нужно убедиться, что LibreOffice установлен и сайт перезапущен.

**В итоговом HTML обнаружены признаки поврежденной таблицы**

Нельзя вставлять результат в Bitrix как готовый. Нужно проверить файл вручную или передать разработчику пример Word-файла.

**В таблицах найдены пустые объединенные ячейки внутри строк**

Это не ошибка конвертации. Конвертер сохраняет структуру Word. Нужно проверить исходный Word: если по смыслу должны быть отдельные пустые ячейки, файл должен поправить инициатор.

**В таблицах найдены ячейки с датой, растянутые на несколько строк**

Это не ошибка конвертации. Конвертер сохраняет `rowspan` из Word. Нужно проверить бизнес-правило: должна ли дата быть общей для группы строк или отдельно указана для каждой акции.

**В таблицах найдены даты, после которых в том же столбце идут пустые ячейки**

Это не ошибка конвертации. В Word дата стоит только в одной строке, а ниже в столбце срока оставлены пустые ячейки. Нужно уточнить у инициатора, относится ли дата к этим пустым строкам или сроки должны быть заполнены отдельно.

## Почему Нужен LibreOffice

Старые `.doc` плохо и нестабильно конвертируются системным macOS `textutil`. В сложных таблицах он может закрыть `</table>` раньше времени и оставить `td/tr` вне таблицы.

Текущий надежный путь:

```text
.doc -> LibreOffice -> .docx -> OOXML parser -> clean HTML
```

Так сохраняются:

- таблицы;
- объединенные ячейки;
- цвета правок;
- автонумерация Word.

## Ограничения MVP

- Инструмент не публикует материалы в Bitrix автоматически.
- Инструмент не создает новые разделы и уроки Bitrix.
- Инструмент не проектирует интерактивные блоки `.js-fancyMyText` из произвольного текста Word.
- Инструмент готовит HTML для вставки в существующий урок.
- После конвертации нужен визуальный контроль человеком.
